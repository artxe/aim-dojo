# Agent Notes

Vanilla JS SPA. `docs/` deploys as-is. No bundler. tsc `checkJs:true`; types in `private.d.ts`. `docs/lib/` excluded from lint/tsc. Package manager is **pnpm** (`pnpm-lock.yaml`) — always use `pnpm run <script>` / `pnpm dlx <pkg>`, never `npm run`/`npx`.

## Where to look first
- Modes / scoring / 2D-3D conversion → `docs/js/game_mode/`
- Input / settings / HUD / DOM / game-sens UI → `docs/js/controller/`
- Camera math / 2D canvas / WebGL → `docs/js/render/`
- Worker protocol / tuple sync / default BG video worker → `docs/js/worker/`
- Sensitivity / DPI numerics / import boundaries → `docs/js/calc/`

## Source of Truth (don't duplicate in notes)
- `state.js` — cross-cutting + localStorage-hydrated state; exports `create_queue()` impl and the shared `shoots_pool`/`impacts_pool`/`impacts_3d_pool` (entry-object pools)
- `pool.js` — `create_pool(make)`: free-list object pool (`obtain`/`recycle`/`clear`) used to keep the per-frame hot path allocation-free
- `bg_store.js` — IndexedDB CRUD for uploaded background-video blobs (split `bg_videos` meta + `bg_blobs` stores; DB version = schema version, NOT app version). `state.bg.video_id` references a record id; blobs never touch localStorage
- `private.d.ts` — global JS types
- `constants.js` — immutable tuning; no imports
- `component/select.js` — state-free light-DOM `x-select` custom element: wires direct `button`/`ul`/`li[data-value]` markup, keyboard/focus/ARIA, native-like silent `.value`, user-only `change`, and `selectaction`/`selectremove`; click-css styles the real child nodes directly
- `controller/dom.js` — all DOM refs/helpers
- `main.js` — startup wiring
- `docs/index.html` — markup plus click-css style/class syntax; `docs/lib/click-css.min.js` expands it at runtime
- `game_mode/<name>.js` — that mode's runtime AND persisted state (module-local). Each mode owns its `STORAGE_KEY = "<name>#best_score"`, `best_score`, score-display init, and exports `clear_best_score()` for `setting.js`. `state.js` knows nothing about modes.

## Conventions
- Read: `const { x } = state.foo`; write: `state.foo.x = v` (style enforced by review, not lint)
- Alphabetize within local groups (imports, object keys, fn defs, independent consts/branches, order-insensitive assigns)
- `constants.fov.<game>` keys use in-game scope names (`x1`, `operator2_5`, `awp2`), never generic `ads`. One `convert_deg_across_aspect` per value — chained conversions fold by composing tan-ratios (e.g. 4:3-horizontal → vertical is `(deg, 4, 3)`, not 4:3→16:9→vertical); no `render_*` intermediate consts
- Comments = JSDoc + narrow ESLint directives only; no prose comments
- State-free import closure (lint-enforced): `eslint.config.js:state_free_files` lists the docs/js modules whose import chain never reaches `state.js` (localStorage → browser-only); per-file `no-restricted-syntax` lets each import ONLY from that set (`calc_sens.js` + both `worker/*_worker.js` keep stricter allow-lists). A new state-free module must be added to `state_free_files` before any state-free file may import it
- Bug-zero code: no defensive error handling, logging, or watchdog instrumentation. Native traces (error site + stack) are the only debugging needed. The ONLY `throw`s allowed are dev-guards on unreachable branches (`else { throw Error(fn) }`, `if (!mode) throw Error()`) and recovery that is genuine feature logic (e.g. `QuotaExceededError` handling, `try/finally` resource cleanup). No reactive error handlers (`onerror`, `"error"` listeners) — the browser surfaces native errors itself
- HTML style/class syntax in `docs/index.html` uses click-css; consult https://raw.githubusercontent.com/artxe/click-css/refs/heads/main/README.md ONLY when genuinely unsure of the syntax — for edits reusing patterns already visible in the file (removing a token, editing a `:not(a,b,c)` list, deleting a `selector/prop=value` rule), just make the change
- Optimization (esp. GC avoidance) exists to IMPROVE runtime perf — never trade perf to achieve it, and don't change documented core APIs (e.g. `create_queue`) for it. Only add hot-path machinery (object pools, etc.) when it is O(1) ultra-light AND removes a real per-frame allocation; skip it where bookkeeping costs more than it saves or where it breaks correctness (e.g. `writing.lines` is consumed raw via `.array`, so flattened but NOT pooled)
- Type cast pattern: `/** @type {T} */(x)/**/` — trailing `/**/` is required for parser stability
- JSDoc types that mirror an existing declaration use `typeof` on the source of truth (e.g. `/** @type {typeof state.stats.shoots} */` for `aim_booster`'s module-local queue) — never re-spell the shape inline
- Arrow functions are OPTIONAL for one-off/disposable use (e.g. a thin wrapper that just forwards args to another function) — allowed, not required. Reused/recycled functions (named, called from multiple sites, hot-path) MUST avoid arrow — use `function` declarations/expressions

## Non-obvious invariants
- Pointer lock and fullscreen are REQUIRED to start a game. `start_game()`'s `.init()` + first `requestAnimationFrame` sit inside the same `try` as the two `await`s, so a rejection from either (e.g. the ~1.25s pointer-lock cooldown right after an Escape release) aborts the whole call: `catch` calls `exitPointerLock()`/`exitFullscreen()` and resets `state.game.mode = null` without ever starting a frame loop. This is intentional — do not "fix" it into a lock-optional fallback.
- Pointer lock loss in-game → auto `stop_game()` (`on_pointerlockchange`)
- `update_camera_view()` ALSO derives `state.camera.sens` (cached `calc_rad_per_px`) and `state.impact.rad_size = px_to_rad(constants.impact.px_size)` at the end (AFTER `dist` is scaled — `px_to_rad` includes the TPP size compensation). Every new `sens` branch must keep these derivations working. `state.camera.dist` is reset to `0` up front; a `tpp`-dimension branch must set it to that game's `constants.tpp.<game>`, and the end of `update_camera_view()` scales it ONCE by `constants.tpp.render_dist_scale` (×10: game-ish tpp units → sky-sphere world units, targets sit at `sky_sphere_radius` 100) before `render/renderer_3d.js:prepare_3d_view()` uses it for the orbit camera — branches always assign the RAW `constants.tpp.<game>`, never pre-scaled. Sens is view-independent — `constants.tpp.<game>` is renderer-orbit-only and never enters a sens formula (see `calc/CLAUDE.md`)
- `state.camera.sens` (rad/px for mouse) is also refreshed in `render.resize_3d()` — width changes there don't pass through `update_camera_view`
- `on_resize()` does NOT call `update_camera_view()`. `state.camera.fov` (horizontal, for sens) is always the game's fixed 16:9-basis value — never derived from the real window aspect — so `game_mode/twitch.js`'s `convert_deg_across_aspect(fov, width, height)` 3D-spawn-spread call reads the CURRENT real `width`/`height` against a FOV that itself never reflects that same real aspect; only a dimension flip (`update_camera_view()`) refreshes `fov` at all, and even then to the same 16:9-basis number (constant-span uses like `val`'s 2.5× ADS zoom don't vary with aspect either way). `state.camera.vfov` (vertical, feeds `render/renderer_3d.js:prepare_3d_view()`'s projection matrix directly) comes straight from a per-game precomputed constant, `constants.fov.<game>.render` (an array of vertical-FOV-at-16:9 numbers — 1 entry for a game with no zoom state, 2 for `[base, zoom]`, `pubg` uniquely has 3 `[fov103, fov80, zoom]`) — it never reads `width`/`height` at all, so it has no resize/staleness concern to begin with. `state.camera.fov` (horizontal) is then derived FROM that vertical value via `convert_deg_across_aspect(vfov, 9, 16)` (a fixed 16:9⇄9:16 swap, not a real-aspect conversion) in `update_camera_view()` itself.
- `state.input.key_space` 30 fps cap works by SKIPPING the rAF body and re-queueing — not by throttling rendering inside the frame
- `state.timer.now_s` comes from AudioContext (`sfx.js:get_audio_time()`), not `performance.now()`. Use it for SFX-aligned timing; `now_ms` for everything else.
- `state.timer.prev_ms` is the prior frame's `now_ms` — tracking modes use it as the start timestamp `s` when pushing to `state.stats.shoots`
- GC avoidance (perf): from `init()` to `dispose()` the per-frame hot path allocates nothing. Queue entries (`shoots`/`impacts`/`impacts_3d`) and per-mode targets come from `pool.js` free-lists — push obtains + fills all fields, the drop site `recycle()`s before `drop()`, and `dispose()` (via `reset_run_state()` or the mode) `clear()`s the pools to release memory. `convert_target_*` write into a passed `out`. `aim_booster` deliberately does NOT pool its module-local `shoots` (no `check_stats`/drop path — it truncates `shoots.array` for the rewind, so there is nothing to recycle)

## Hot paths
- Start: `controller/menu.js` passes the mode name to `logic.start_game(mode)` (which sets `state.game.mode = mode`) → request lock+fullscreen → `mode.init()` → first rAF
- Frame: `logic.on_frame()` → space-hold gate → timer/audio update → `mode.on_frame()` → if `now_ms >= state.hud.next_update_ms`: `check_stats?()` + `update_hud()` → `mode.render()` → next rAF
- Resize: `controller/window.js:on_resize()` → `resize_2d()` → `resize_3d()` (no FOV refresh)
- Dimension flip (RMB down/up, mousecancel): `mode.update_dimension()` → `update_camera_view()` → `convert_camera_to_2d/3d` if camera state non-zero → `convert_target_to_2d/3d` → convert in-flight lerp endpoints
- Input: `controller/window.js` writes `state.input.key_*/mb_*`, calls `mode.shoot()` (Q/W/E/R/A keydown, LMB down) or `mode.update_dimension()` (RMB down/up if defined, else `shoot()`)
- Worker: calc calls use `post_calc_worker_message({ fn, ...args })`; responses are `[fn, ...payload]` and `manager.js` dispatches directly. BG uses separate object messages via `post_bg_worker_message()`

## State landmarks
Split: `state.js` = persisted-or-cross-cutting only. Mode runtime = module-local in `game_mode/<name>.js`.
- `state.game` — active `mode`, `resolution`, `sens` (game), `raf_id`, `rest_timeout` (screensaver setTimeout id), `width`/`height`/`dpi_scale`/`lol_sens`/`pubg_fov`
- `state.camera` — `dimension: "2d"|"fpp"|"tpp"`, 2D `x/y`, 3D `yaw/pitch`, `fov` (horizontal, 16:9-basis, paired with `width` for `sens`; derived from `vfov` via `convert_deg_across_aspect(vfov, 9, 16)` in `update_camera_view()`), `vfov` (vertical, 16:9-basis, read straight from `constants.fov.<game>.render` — fed straight into `prepare_3d_view()`'s `mat4.perspective`), `dist` (TPP camera offset behind pivot in world units, `constants.tpp.<game>` × `constants.tpp.render_dist_scale`; reset to `0` at the top of every `update_camera_view()` call, set only by `tpp` branches, scaled once at the end), `width/height`, `sens` (cached rad/px, derived from fov+width), WebGL `proj/view` (mutated in-place by `prepare_3d_view()`)
- `state.stats` — shared 30s scoring window (`constants.hud.window_ms`); modes push to `shoots`; `check_stats()` expires + decrements aggregates. Exception: `aim_booster` pushes to its own module-local `shoots` queue (rewind history survives other modes' `reset_run_state()`)
- `state.impact` — `rad_size` only (3D, derived by `update_camera_view()` as `px_to_rad(constants.impact.px_size)`, reset by 3D-capable mode `dispose()`); 2D impact radius reads `constants.impact.px_size` directly
- `state.impacts` / `state.impacts_3d` — visual queues; cleared per-dimension on `dispose()`
- `state.timer` — `now_ms` (rAF), `now_s` (AudioContext), `prev_ms`, `start_ms`, `next_frame_ms` (space-hold gate), `fps`
- `state.input.key_a/e/q/r/space/w, mb_left/mb_right` — raw flags; modes read in `on_frame()`/`shoot()`
- `state.bg` / `state.crosshair` / `state.dpi_norm` — user-settings containers, localStorage-backed (key = state-tree path). `state.bg` is `{ type: "default"|"video"|"youtube", video_id, youtube_link }` only; uploaded video blobs live in IndexedDB via `bg_store.js`
- Mode-private state (`best_score`, peak/targets/lerps/timers) lives in `game_mode/<name>.js` module-locals, NOT in `state.*` (see `game_mode/CLAUDE.md`). localStorage key for module-private fields uses `<module>#<field>` form

## create_queue (state.js)
Logical-head ring. `.length`, `.at(i)`, `.push()`, `.drop()`, `.clear()` respect head. `.array` is the RAW backing buffer (head-dropped entries still present) — only use when passing to worker TOGETHER WITH the head offset (`lines_start = array.length - length`; a structured-cloned plain Array knows nothing about the head, so the worker must iterate from that offset), or for `aim_booster`'s direct `.array.length =` rewind truncation. Compacts IN PLACE (`copyWithin`, no allocation) when `head > 2048` AND `head > q.length/2` (HALF THE RAW BACKING ARRAY, not the head-respecting logical `.length`) — keeps the backing linear (so `.array` and `.array.length =` stay valid) while avoiding a periodic `slice` allocation spike.

Entry objects are pooled OUTSIDE the queue (`pool.js`), not by `create_queue` itself — push a pooled object, `recycle()` it at the drop site. A queue whose `.array` is consumed raw elsewhere (e.g. `writing.lines` → worker) needs the consumer to skip the pre-head region (see `lines_start` above). `writing.lines` flattens its `Line` to `{ ex, ey, sx, sy, t }` (no nested point objects, no aliasing) but stays unpooled — the per-check structured clone dwarfs the stroke-literal allocation.
